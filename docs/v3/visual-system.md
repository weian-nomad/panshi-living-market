# 《盤勢・眾生》V3 視覺與動態系統

_2026-07-20｜供 Figma、生成圖、前端動態與短影音共用_

## 畫面要像一個有人住過的觀象所

黑墨是空間，暖紙承接人物，古銅留下時間。真實市場資料只用一個冷色訊號。畫面可以神祕，但時間、來源、狀態與操作必須一眼看懂。

使用者提供的參考圖保留三件事：高密度人物觀察、紙本與深色面板交錯、命盤和關係網是人物內在的視覺語言。V3 拿掉滿屏儀表板感，讓公共世界與人物場景先出現。

## 三層視覺

### 世界層：一個會自行前進的 2.5D 街區

眾生場是一幅可探索的橫向場景。觀象所、公司大樓、住家、咖啡店、車站與醫院以紙雕、墨線和克制景深組成。首屏鏡頭只跟一齣今日主戲，人物在場景裡有位置；今日五人與今日五股要滑到第二屏才展開。事件用光、聲音和鏡頭切換顯示，不用滿天 icon。

城市不是自由行走的 3D 地圖。第一版只做一個完成度高的街區，桌面可平移，手機用垂直事件鏡頭。每個地點都能回答「誰在這裡、發生了什麼、和市場哪一件事有關」。

### 人物層：肖像、房間與內心戲同屏

人物觀測室沿用參考圖的精神：左側是有環境線索的半身肖像，中央是此刻內心戲和注意焦點，右側是命盤、心理需求與狀態；下方展開證據、關係、生活負荷和時間線。

卡片不排成十個同重量模組。首屏只保留一張主肖像、一句尚未解決的念頭、一段關係變化、三項注意線索和一個館主動作。

### 證據層：像研究檔案，不像交易終端

價格、公告、來源與 cutoff 使用固定網格和高對比字體。紅綠只描述方向，不代表人物好壞。象徵解讀使用銅色，真實市場資料使用冷色，虛構人生使用紙色；每個 truth class 同時有文字標籤與圖形，不只靠顏色。

## 字型不是換一套 web font

品牌需要兩套字：一套可長時間閱讀，一套只屬於《盤勢・眾生》。

| 用途 | 決策 |
| --- | --- |
| 品牌字標 | 為「盤勢・眾生」六字另畫字形。筆畫取自宋體骨架、碑刻收尾和市場跳動的窄缺口，只用於字標與片頭。 |
| 章節與人物名 | 使用授權清楚的繁中明體作 master，再為 40–60 個高頻標題字設計 alternate glyph。字面偏窄，保留紙本呼吸。 |
| 介面與長文 | 使用可讀性高的繁中黑體，數字不混入仿古字。字重只用 Regular、Medium、Semibold。 |
| 時間、價格、版本 | IBM Plex Mono；開啟 tabular numbers，讓同欄位不跳動。 |
| 短箋手寫 | 只使用一套授權確認的手寫字，限 18 字以內；正文永遠不用手寫體。 |

正式上線不從第三方字型服務即時載入。保留 font file、版本、license、subset 清單與 fallback 截圖。自製字形要能在高對比、放大 200% 和 Windows fallback 下維持意思。

## 生成圖資產

生成圖負責「畫面有生命」，正式資料負責「這個人是誰」。圖像模型不能發明年齡、家庭、職業、情緒或關係事件。

### 人物 master

- 成人編輯插畫。保留真實年齡、皮膚和生活痕跡，不做通用漂亮臉。
- 版畫墨線、礦物顏料、紙纖維；光線像窗邊，不像遊戲抽卡。
- 每人固定輪廓、髮型、配色、常用物件和五個表情 anchor。
- 保存 4:5 主圖、1:1 頭像、9:16 分享裁切與透明安全區。
- 分出臉、眼、呼吸、衣料、前景、背景與命盤光紋七層。動態由前端控制，不逐幀重新生成。
- 每張圖保存角色 brief、prompt、negative prompt、生成識別、日期、核准人、用途與 alt text。

### 世界 scene plate

首批只生成六個地點：觀象所、公司研究室、住家、咖啡店、車站、夜間屋頂。每個地點有晨、行市、收盤、夜間四種光線，但使用同一透視與物件位置，避免世界每天變形。

場景輸出至少包含遠景、建築、室內主體、人物行走面、前景遮擋、光影與事件特效層。桌面 16:9、手機 9:16、分享 1:1 共用同一構圖安全區。

## 動態有五種，不能無限加

1. **世界呼吸**：8–12 秒低幅循環。窗光、紙纖維、遠景人流緩慢變化；角色沒有事件時不做誇張 idle。
2. **市場落地**：新事實進入時，冷色線從來源移到受影響地點，400 ms 內完成。不可用光線暗示漲跌吉凶。
3. **注意轉向**：人物注意到事件時，眼神、桌上物件和三項線索改變；先動物件，再動文字。
4. **收盤回放**：拖動時間時，畫面只改當下節點。事實、記憶、關係和行動依序顯影，總過場不超過 1.2 秒。
5. **新生／放野／重聚**：紙頁打開或收進公共世界，最長 1.8 秒。結果由伺服器狀態決定，動畫不能假裝成功。

Reduced motion 取消位移、縮放、景深與視差，只保留 120 ms 內的 opacity、stroke 和文字狀態。動畫被中斷時，畫面仍要顯示正確世界狀態。

## 互動手感

- 世界層：拖動畫布或滑動事件鏡頭；滾輪縮放只在桌面提供，且有鍵盤替代。
- 人物層：短箋、卷宗、邀請函、鑰匙與時間是可拖動物件。每次拖動都要先顯示可能影響與不可控制的事。
- 回放層：一條可抓取的時間線；鍵盤左右鍵逐節點移動，螢幕閱讀器讀出節點時間、truth class 和結果。
- 關係層：桌面用星圖，手機預設用排序列表；使用者主動切換才顯示縮小星圖。

## 禁用語法

- 滿版 K 線、即時漲跌跑馬燈、買賣按鈕、券商紅綠終端。
- 霓虹 AI 漸層、玻璃卡、漂浮粒子、金幣、寶箱、轉盤、倒數焦慮。
- 通用動漫抽卡、幼齒 chibi、臉上貼星座符號、命盤線連到股價方向。
- 每個面板都等寬等高；每個人物都用同一姿勢；每個事件都彈通知。
- 用長動畫遮住資料延遲、修正、作廢、訂閱失效、恢復權益或席位衝突。

## Figma handoff gate

每個 frame 要附：世界時間、資料 truth class、玩家此刻能改什麼、下一個不可逆時點、loading／empty／stale／held／void／offline 狀態、鍵盤／screen-reader／reduced-motion 等價操作。沒有這些註記，畫面再漂亮也不能交工程。
