## Figma

- 定義「今日主戲第一段完成」的狀態；其他人物與今日五股在此之前不得出現。
- 用實際原型驗證觀眾能否在十五秒內理解主角、私人利害、已做選擇與未揭曉後果。
- 畫全照拂時機的七個狀態：可介入、已介入、明確不介入、逾期失效、等待後果、已揭露、可回放。「不介入」不能長得像錯過。
- 每個介入要先讓人看見確定代價，不顯示推薦、成功率或暗示答案。
- 分開呈現正常照拂、寬限期、公開狀態、可回席、席位衝突、主動放手。主動放手不能和失訂轉公開混用。
- 三個公開判斷必須嵌在人物敘事與因果帶中，不能長成持倉表或績效卡。
- 思考核心只呈現取捨，不形成強弱排序或付費暗示。

## 技術架構

- 先定義唯一權威的 `WorldNode`。照拂時機的開啟、失效、處理、結算與後果揭露都對齊同一套節點。
- 主動放手、失訂轉公開、寬限期與回席是不同狀態轉移；七日限制只屬於主動放手。
- 每個照拂時機只能處理一次。「介入」與「明確不介入」都要不可覆寫、不可重試。
- 公開判斷建立後不可修改，只能追加後果；對人物狀態的影響需要上下限。
- 主戲編排層只能選擇與排序既有事件，不得改寫人物行為來湊戲。
- 共同場域要能產生跨人物因果，不能靠隨機相遇或事後拼接。
- 換核不能重置記憶、關係、公開判斷或世界狀態。
- 眾生場、決策回放台與市場證據頁必須引用同一版本的 truth classes 與事實封存。

## 法律與資料

- 取得並記錄行情、公告、重大訊息的展示、延遲與再利用權利。
- truth class、來源、cutoff、版本、修訂與留存責任要可稽核。
- 公開判斷與標準化模擬部位不得形成個人化操作指令。
- 人口統計、命盤、職業與壓力只約束虛構角色，禁止回推出真實個人。
- 生成肖像不得近似真實人物；分享內容需有清楚的使用與版權規則。
- 訂閱頁要說清楚照拂關係、公開狀態、回席資格與主動放手的差異。
