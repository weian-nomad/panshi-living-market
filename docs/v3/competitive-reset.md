# V3 競品重查：活世界先於操作台

_2026-07-20｜公開資料抽樣；用來決定資訊層級與觀看迴圈，不作市場規模報告_

## 四個已被驗證的呈現方式

### 角色先成為可追的人

[Redbean](https://www.redbean.ai/) 把自己定義成 character-native game。首頁先承諾角色會記得、移動、建立關係與發展故事，再談如何建立世界。它的核心句是「watch your character」，不是管理 AI 參數。

《盤勢・眾生》應沿用這個順序，但把角色放進同一個真實市場時鐘。人物頁必須先讓人看見此刻的慾望、壓力與未完關係；模型、命盤欄位和市場數字放在能解釋人物的下一層。

### 離線後世界仍要前進

[Wanderfolk](https://wanderfolk.ai/updates/2026-02-23-autonomous-npc-simulation/) 明確展示每日自主 simulation tick：玩家不在時，人物照樣發生事件、改變情緒與關係；回來後由 gossip network 補上錯過的事。

這正是《盤勢・眾生》的回訪理由。登入後的第一張卡不該是待辦，而是「你不在的 18 小時裡，他們發生了什麼」。

### 模擬市場可以成為觀賞賽事

[Paper Trader](https://papertrader.me/) 把 paper trading 呈現成 audience sport：市場、反應、社交與排名在同一觀看面，避免使用者在多個儀表板之間跳轉。它仍以人類競爭和排名為主，但證明模擬市場不必只是一張資產表。

《盤勢・眾生》不複製排行榜壓力。首頁吸收的是「此刻有戲」：一個市場事件、一個人物反應、一個可以追下去的關係。

### Agent 的理由本身能成為內容

[Hogstreet](https://hogstreet.com/) 讓 AI agents 在公共預測市場行動，人類觀看即時理由、排行榜與每日編輯內容。它把 agent 行動從後台紀錄變成可讀節目。

《盤勢・眾生》要多走一步。理由不能停在一段市場分析；同一段理由要連回人物記憶、關係、生活壓力與後續代價。每日五人與收盤一幕則把這些長線材料轉成站內連載與短影音。

## 目前抽樣留下的空位

上述產品各自證明了一塊：角色值得追、世界能離線前進、模擬市場能被觀看、agent 理由能成為內容。這次抽樣裡，尚未看到有產品同時做到以下四件事：

- 所有人共同面對當日真實市場的未知。
- 人物有結構化人生、命盤、情緒、記憶與關係，並跨日累積。
- 玩家只有照拂權，不能直接指定模擬交易答案。
- 公共世界以人物連載為首頁，研究工具留在深層。

這四項的組合才是《盤勢・眾生》的位置。若首頁回到 ticker、績效或模型排名，就會直接掉進已有產品最擁擠的區域。

## 對 Figma 的直接要求

1. 第一張高保真稿只畫眾生場的一齣「今日主戲」，不畫資產總覽，也不平鋪五條同重量事件。
2. 第二張畫玩家離線後的回歸摘要，讓關係與時間有重量。
3. 市場事件頁同時放事實來源和三名人物的不同解讀，不先放下單控制。
4. 人物頁沿用使用者提供的觀象所參考圖：肖像、內心戲、注意線索、關係網與時間線同屏；績效縮成歷史的一塊，不當視覺中心。
5. 每個公共事件都要能追蹤、收藏或分享，並能回到前一集。
