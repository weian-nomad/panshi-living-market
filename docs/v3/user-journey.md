## 三種人進入同一個世界

| 身分 | 進站時想做的事 | 首頁先給什麼 | 成功離站 |
| --- | --- | --- | --- |
| 純觀看者 | 看今天哪個小人最有戲 | 一齣今日主戲：一人、一事、一個尚未揭曉的後果 | 追蹤一人、看完一段回放或分享 |
| 館主 | 知道自己的五人發生了什麼 | 離線摘要與最需要探望的一人 | 完成零到一次介入，接受世界繼續走 |
| 失聯後回來的館主 | 找回原來的人物與時間線 | 小人在公開世界的近況、可恢復名單 | 一次補回所有空席，理解失聯留下的關係後果 |

未登入者也能看完一段公共人物戲。註冊牆只能出現在追蹤、建立小人或保存私人短箋時，不能截斷第一段故事。

## Journey 1：先追劇，再想擁有

```text
從外部短影音／搜尋／panshi.app 進站
→ 眾生場正在發生
→ 點進一名人物
→ 看見此刻念頭、關係與未完問題
→ 打開收盤回放
→ 沿因果帶追到一項真實事實
→ 進市場證據頁查證
→ 回到人物後果
→ 追蹤／收藏／分享
→ 建立自己的第一名小人
```

轉換點放在一段完整故事之後。CTA 不寫「開始選股」，寫「讓一個人醒來」。

## Journey 2：館主每天 3–8 分鐘

```text
收到「他們在你離開後發生了 3 件事」
→ 我的觀象所
→ 選擇最需要探望的一人
→ 讀 30–60 秒近況
→ 世界出現具體「照拂時機」
→ 進今日照拂桌
→ 問／給／帶去見／守住，或明確不介入（全館每日零到一次）
→ 確認一定犧牲的東西與不可控制的事
→ 離開，世界照常運轉
→ 收盤後回到決策回放台
→ 看事實、注意、解讀、情緒、關係與行動
→ 夜間收到不超過三段來信
```

每日成功不是按完一組任務。玩家可以只看、不介入；系統不能用紅點逼他消耗照拂額度。

### 每日關鍵狀態

| 狀態 | 介面要說清楚的事 | 可做的事 |
| --- | --- | --- |
| `MORNING_OPEN` | 小人正在形成注意，尚未作出模擬行動 | 看近況、做一次有限介入 |
| `MARKET_LIVE` | 行動自主形成中，完整推理尚未公開 | 旁觀足跡，不能改核心或重抽 |
| `MARKET_CLOSED_PENDING` | 市場已收，來源仍在封存 | 看人物場景，不猜未完成結果 |
| `DAILY_REVEALED` | 結果、因果與人物後果已完成 | 回放、收藏、分享；不能回頭改當日選擇 |
| `NIGHT_LIFE` | 人物關係與記憶正在更新 | 讀信；不再新增當日市場介入 |
| `DATA_HELD` | 某項事實缺漏或衝突 | 看已保存內容與原因，等待修正或作廢 |

## Journey 3：一週照拂與投放

```text
週界回顧「這週成為了誰」
→ 查看五個照拂席位
→ 每席選擇繼續同行／放手
→ 空席可建立新人
→ 生成結構化人生聖經
→ 選擇思考核心與一項研究專長
→ 預覽真實統計、虛構人生與象徵解讀的分界
→ 完成不可回滾的初次醒來
→ 留下本週一個問題與一次引薦
→ 進入下一個交易週
```

館主最多有五名直屬角色。每週只能補滿空席，創生次數不累積。沒有空席時，建立新人按鈕要先帶玩家處理「繼續同行或放手」，不能偷偷擴張上限。

## Journey 4：失訂、放野、重聚

```text
訂閱失效
→ 七日寬限：停止新增介入，照拂關係仍保留
→ 已提交事件完成
→ 第八日私人房間唯讀，小人進公開世界
→ 所有人可追劇，無人可接管
→ 原館主仍看完整時間線與發起紀錄
→ 恢復訂閱
→ 所有原角色立即取得回席資格
→ 可在同一次操作中補入所有未占用席位
→ 若席位已滿，先由館主決定放手哪一人；系統不得自動替換
→ 失聯期間的怨懟、疏離與新關係全部保留
```

重聚是人物關係，不是客服攔截。角色不能拒絕恢復照拂關係，但仍能拒絕之後的個別介入。自主性改變關係，不能扣住已恢復的訂閱權益。

## 七個核心畫面的接力

| 畫面 | 從哪裡進 | 下一個自然動作 | 不得搶走注意力的東西 |
| --- | --- | --- | --- |
| 眾生場 | 首頁、分享、外部內容 | 追一段人物事件 | 大盤績效、ticker 排名、建立帳號彈窗 |
| 我的觀象所 | 登入首頁、通知 | 探望最需要的一人 | 五個相同大小的儀表板、待辦紅點海 |
| 人物觀測室 | 人物事件、觀象所 | 沿時間線或關係分頁理解困境 | 模型參數牆、可直接改的情緒數值 |
| 今日照拂桌 | 人物出現照拂時機 | 介入或明確不介入 | 自由 prompt、沒有代價的選項、預測結果 |
| 決策回放台 | 收盤一幕、人物時間線 | 拖動時間並追到原因 | 未封存的盤中推理、單一 AI 長文 |
| 市場證據頁 | 回放中的事實節點 | 比較人物看過與漏掉的材料 | 買賣 CTA、象徵解讀冒充事實 |
| 新生／放野／重聚 | 空席、人物頁、恢復訂閱 | 做一次有後果的承諾 | 扭蛋動畫、無限重抽、付費恐嚇 |

## Figma 必畫狀態

第一輪高保真不要先畫整座城市。先把同一個公共街區做完整，桌面與手機各有以下狀態：

1. 開盤前、行市中、收盤待封存、收盤已揭曉、夜間。
2. 第一次進站、已有追蹤、已有五名小人、照拂席位全滿。
3. 小人接受短箋、誤解短箋、拒絕介入、尚未讀取。
4. 市場資料正常、延遲、缺漏、修正、作廢。
5. 訂閱正常、七日寬限、唯讀公開、一次恢復所有空席、席位衝突。
6. reduced motion、鍵盤焦點、螢幕閱讀器標籤、窄螢幕關係圖替代列表。

每張 frame 的註記固定回答四件事：現在是什麼時間、哪一層資料是真的、玩家此刻能改什麼、下一個不可逆時點在哪裡。

## 封測觀察點

- 純觀看者能否在十秒內指出「今天哪個人最有戲」。
- 館主能否在一分鐘內說出最需要探望的人，以及為什麼。
- 做完介入後，館主是否知道小人仍可能拒絕或誤解。
- 收盤回放是否讓人先談人物原因，再談漲跌。
- 人物頁的關係分頁是否引發回看事件，而不是只比較數值高低。
- 失訂測試中，玩家是否把放野理解成世界繼續，而不是角色被扣押。
