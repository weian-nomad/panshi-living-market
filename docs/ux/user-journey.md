## 三段旅程

### 1. 第一次進場到回合結算

`首次進場 → 領取五位角色 → 選擇可用內容域 → 選 20 家公司 → 編五份卷宗 → 開啟五交易日回合 → 每日排席 → 封席 → 揭曉 → 因果 → 反事實 → 回合結算`

第一次沒有完成卷宗時，使用與績效無關的 deterministic starter config。當期真實公司內容域只有在外部 gate 完整通過後才存在；歷史隔離模式不能在事後轉成當期模式。

### 2. 每日 60 秒排席閉環

`昨日關鍵因果 → 拖五人 → 拖五卷宗 → 合法行為預覽 → 儲存 → 08:25 封席 → 角色自主研判 → 收盤結算 → 逐席揭曉 → 單變量反事實`

封席前的預覽只回答「這個安排允許什麼」，不能洩露角色會選哪家公司、方向、信心或預期結果。錯過 cutoff 時沿用上一個有效交易日的排席，不由 AI 最佳化。

### 3. 掌控權與異常

訂閱失效後，五名角色在回合邊界進入公開自主生活；其他人只能延遲觀看。恢復後，原玩家在下一個回合邊界取回同一五人，不回檔、不重生。

同一張旅程圖也固定以下狀態：

- 休市不消耗回合、脈絡或風險預算。
- 資料延遲不得先猜結果；無法補齊時單席或全 session 作廢。
- `qmax=0` 鎖桌位，不可藉換人或換卷宗重置。
- 全席鎖定進入恢復期，仍可看人物狀態與準備下一回合。
- 離線只保存本機草稿；封席與 canonical placement 以伺服器接受的版本為準。

## 首屏十秒測試

使用者在十秒內必須能回答四件事：

1. 今天還能不能排席。
2. 哪位角色保有脈絡。
3. 哪張桌位受到風險煞車。
4. 昨天哪個因素真正改變了行動。

若需要進第二層畫面才能回答，首屏失敗。若使用者先注意到行情、ticker 或「哪隻會漲」，產品方向失敗。

## 原型驗收路徑

| 路徑 | 必測行為 | 成功訊號 |
| --- | --- | --- |
| 新手首日 | 不讀教學完成 starter config 與封席 | 90 秒內完成，知道角色才做最後決定 |
| 第三日 | 在脈絡與三項新事實間做取捨 | 能說出保留或拆散某組合的理由 |
| 鎖席日 | 一席 `qmax=0` | 不嘗試以換人洗掉風險結果 |
| 全部不做 | 五人均選 `NO_ACTION` | 接受為有效結果，不尋找重抽按鈕 |
| 資料延遲 | settled 時間未知 | 理解「等待資料」和「作廢」的差別 |
| 失訂 | 回合結束後失去排席權 | 理解角色仍活著，但不能被他人接管 |
| 恢復 | 下一回合邊界取回角色 | 能辨認人物歷史連續且未回檔 |

## 畫面推導順序

高保真稿依下列順序製作，不從首頁視覺直接開畫：

1. 五席圓桌的 desktop 與 mobile 排席狀態。
2. 卷宗編組與規則衝突修復。
3. 封席前合法行為預覽與封席後不可改狀態。
4. 逐席揭曉、因果帳本與單變量反事實。
5. 分區、回合結算與角色長期狀態。
6. 公開自主生活、失訂與恢復。
7. missing、stale、corrected、void、locked、offline 與 reduced-motion。

每張畫面都要標示它對應的 command、authoritative state、可見 projection 與不可出現的資訊。未完成這四項標示的畫面不能進工程 handoff。
