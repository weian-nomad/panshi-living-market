# V4 相鄰產品研究

_2026-07-20｜這份研究只回答互動形式，不用競品功能表反推產品_

《盤勢・眾生》找不到一個可直接照抄的同類。較有用的方法，是拆解六種相鄰產品各自留住人的原因，再把不相容的部分明確排除。

| 相鄰類型 | 可借的東西 | 必須避開 |
| --- | --- | --- |
| [Generative Agents](https://arxiv.org/abs/2304.03442) 與 [AI Town](https://github.com/a16z-infra/ai-town) | 共享世界、記憶、計畫與社交傳播會產生事先沒寫好的群體事件。 | 對話紀錄本身很快變成噪音；不能把「很多 agent 在聊天」當成可看性。 |
| [RimWorld](https://rimworldwiki.com/wiki/About_RimWorld) | 人物需求、關係和外部事件相撞後，故事由玩家自己記住，不需要每日官方劇本。 | 管理殖民地、資源最佳化和大量狀態面板會把本產品拖回策略工具。 |
| [Dwarf Fortress](https://store.steampowered.com/app/975370/Dwarf_Fortress/) | 長期正史和細小規則能讓普通事故有份量。 | 深度若只藏在資料庫裡，觀眾看不見；複雜度不能替代鏡頭與表演。 |
| [Neko Atsume](https://www.nekoatsume.com/sp/en/about.html) | 世界在離線時繼續，回來看看誰出現本身可以成立。 | 收藏、餵食和稀有度會把居民變成可取得的物件。 |
| [Rusty's Retirement](https://store.steampowered.com/app/2666510/Rustys_Retirement/) | 一眼看見許多小角色同時做事，低干擾也能形成陪伴。 | 產量、升級與自動化經濟不是本產品的留存理由。 |
| [MarketDraft](https://marketdraft.com/) 等 fantasy stock game | 真實市場提供所有人共同的時鐘和不能預寫的結果。 | 報酬榜、競賽和虛擬資產會讓人物退回投資策略的外皮。 |
| [Fortunara](https://fortunara.app/) 與 [AstroApp](https://astroapp.com/en/main-features-17/financial-astrology) | 玄學語言可以提供每日回訪的節律與個人解讀。 | 訊號、買賣時機和星象指標會造成投顧混淆，也讓玄學變成廉價圖層。 |

## 研究裁決

上一版混合了角色養成、財經工具和事件審計，最後長成一張資訊很全、但沒有「此刻」的 dashboard。這次只保留三件互相相容的事：

1. AI Town 的群體傳播，但以可見動作和空間表演呈現，不以聊天 log 呈現。
2. RimWorld 的長期後果，但不讓觀眾管理人物或最佳化資源。
3. Neko Atsume 的回來看看，但回訪理由是人物改口、關係變化與昨日後果，不是收集。

真實市場是共同天氣，玄學是人物的私人濾鏡，跟拍是觀眾唯一的參與。三者只在同一個現場畫面相遇，不再各自長出一套功能頁。
