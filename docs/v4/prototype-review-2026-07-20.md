# V4 跟拍原型審查

_2026-07-20｜此文件只記錄會影響第一輪產品驗證的裁決，不是功能待辦清單_

## 第一輪裁決

方向「繼續做」，核心動詞保留為跟拍。第一版實拍仍像選取人物的金融監控介面，因此不得拿來測留存。

審查列出的五個 P0 已落地：

1. 跟拍改成有資訊代價的鏡頭推進。主角約佔場景高度 38%，非主角與全景字幕退場。
2. 首訪前八秒說明按住與拖曳；按下後出現 180 ms 進度環，成功一次後永久收起。
3. 刪除替人物讀心的結論，只呈現可見動作、近距可聽語句與有來源的市場事實。
4. 場景改用真實十分鐘播放，不再把十分鐘壓成一百秒。
5. CSS 人偶換成 16 位有獨立輪廓、服裝、姿勢與道具的原創圖層。

架構 P0 也已修正。Web 只把 PointerEvent 轉成世界座標；命中與優先序不再依賴 DOM 排序、`elementFromPoint`、CSS transform 或 React 元件生命週期。

## 第二輪裁決

裁決為「可進 24 人測試」，剩餘 P0 為零。全景選人、近距跟拍、視野資訊封鎖與可觀察字幕已形成可測閉環，不會系統性製造看滿十分鐘或隔日認人的假結果。

每位居民尚未拆成頭、軀幹、雙臂和道具五張 raster，判為 P1。第一輪依附主要靠輪廓、服裝、姿勢、道具、聲音與事件累積；目前每人一張獨立 sprite 已足以測身份辨識。五層拆分只影響後續微動作密度。

100 人與原生 App 沒有剩餘架構 P0。守住的邊界是：世界座標引擎只接純資料，不引用 `HTMLElement`、`PointerEvent`、`DOMRect` 或 CSS transform。擴到 100 人只替換空間索引；原生端只替換輸入與渲染 adapter。

## 測試對象與任務

- 24 位第一次看見產品的成年使用者。
- 不先講產品類別，只請他們在手機上觀看這個十分鐘片段。
- 主持人不示範手勢、不提示主角，也不解釋人物心理。
- 隔天用同一台裝置再次打開，觀察是否會在無提示下先找昨天同一位居民。

## 現場紀錄

逐人記錄第一次成功跟拍時間、第一次拖曳交接時間、前景觀看秒數、首日跟拍時長第一名，以及隔日第一個跟拍對象。能否說出人物矛盾、能否分清公告事實與人物說法仍做診斷紀錄，但不取代行為門檻。
