> **你不下單，你排席。五個有連續人生的小人，封席後會自己做決定。**

## 產品判決

《盤勢・眾生》是一款五人 AI 命運策略遊戲。玩家經營一支會觀察、互相影響、承擔結果，也會因長期安排而改變的隊伍。

玩家每回合編成五份公司卷宗，在接下來五個有效交易日，把五個人與五份卷宗排進五張固定桌位。桌位決定風險、資訊視角和一名同伴來源；封席後，角色依當時可用的證據，以及各自的人格、命盤、記憶、情緒與關係，自主選擇公司、上行／下行／不做及信心部位。揭曉時，玩家看見決定、結果與一個可驗證的反事實。

核心承諾只有一句：**玩家安排人與位置，角色承擔最後選擇。**

## 唯一主動詞：排席

上一版「組盤」同時開放公司池、資訊槽、職責、關係線與思考核心，會把每日玩法做成多欄表單，也讓結果無法歸因。現行版本將這些權力綁進可見的桌位與卷宗。

### 每回合：卷宗編組

每個回合包含五個有效交易日，不以日曆週切割。開始前，玩家從全分區共用、當季凍結的 40 家合格公司 universe 中選出 20 家，編成五份各四家的公司卷宗。40 家公司預先凍結為八個產業 cohort，每組五家。

- 同一家公司只能出現在一份卷宗。
- 每份卷宗的四家公司必須來自四個不同 cohort。
- 全隊 20 家中，同一 cohort 最多三家。
- 卷宗整個五交易日回合固定，不能每日追逐熱門公司。
- 其他玩家直到回合結束後，才看得到精確卷宗與每日座位。
- 思考核心也在回合邊界設定，不進每日操作。

公開首版使用至少一年前、權利明確的歷史資料，遊玩期間以虛構公司名與遮蔽日期呈現。因此「公司」是歷史 episode 中的虛構代稱，不是當下可搜尋的真實 ticker。

### 每日：兩種拖曳

桌面只有兩個動作：

1. 把五個人放到五張桌位。
2. 把五份公司卷宗放到五張桌位。

| 桌位 | 資本／風險 | 固定資訊視角 |
| --- | --- | --- |
| 守門席 | 25%，最低回撤線 | 公司公告 |
| 核心席甲 | 20%，中回撤線 | 產業事件 |
| 核心席乙 | 20%，中回撤線 | 歷史價量 |
| 側翼席 | 20%，高回撤線 | 跨公司比較 |
| 探索席 | 15%，最高回撤線 | 新事件與異常 |

五人圍成固定方向的圓桌：每人先獨立形成初稿，再同時向右手邊送出一份密封主張；每人只收到左手邊一人的主張。玩家不畫社交圖，也不能指定角色是否聽從。

每次交換人或卷宗，介面即時顯示合法行為空間的差異：四家可選公司、最高信心部位、資訊視角與同伴來源。預覽不得顯示角色的公司偏好、方向、信心或預期結果。

### 60 秒流程

| 時間 | 玩家行動 |
| --- | --- |
| 0–8 秒 | 看昨日唯一最關鍵因果：哪個因素改變了哪個人的決定 |
| 8–25 秒 | 交換兩名角色；看桌位、風險與同伴來源如何改變 |
| 25–42 秒 | 交換兩份卷宗；確認每人四個合法標的 |
| 42–52 秒 | 查看今日合法行為預覽 |
| 52–60 秒 | 按下「封席」；08:25 前仍可返回修改 |

每日流程不得出現權重輸入、百分比調參、自由畫線或超過十次點擊／拖曳。

## 學會自己的隊伍

第一天的驚喜應來自可解釋的反轉：被放在探索席的自信角色，因新公告、過往受傷記憶與左手邊的反方主張而選擇不做；平時保守的人取得跨公司比較後，反而建立下行立場。

每名角色每天固定只有三個正式注意槽。前兩個有效交易日，三項都來自當日新事實；同角色、同桌位、同卷宗完成兩日後，第三日起改為兩項當日新事實，加一項由 deterministic salience 選出的未解舊事實。玩家不能挑舊事實，脈絡也不會疊成四項或五項。

換角色、換桌位或換卷宗，當日恢復三項新事實。留下會取得縱向理解，代價是少看一項新事件；換席取得完整新資訊，代價是失去脈絡。脈絡不增加總資訊量。

八回合後的熟練，不是預測股票，而是能預測隊伍：

- 誰在高喚醒時會放大信心。
- 哪類記憶會讓某人迴避特定事件。
- 誰適合先判斷，誰適合成為反方。
- 何時值得犧牲脈絡深度拆散共識。
- 如何用桌位限制一名好角色的壞狀態。

若第三天後，測試者仍只用「股票選錯」解釋結果，排席原型即判死，不靠增加城市、對話或內容量補救。

### 回合狀態與缺席

```text
ROUND_DRAFT → ROUND_READY
→ SESSION_OPEN → SESSION_SEALED → SESSION_SETTLED
→ 完成第 5 個有效交易日 → ROUND_SETTLED → 下一回合
```

- 第一次未配置：使用由帳號 seed 固定、產業平衡且與績效無關的 starter 卷宗與排席；它必須列入盲策略測試。
- 後續回合未配置卷宗：沿用上一回合，不依歷史績效自動換公司。
- 錯過 08:25：沿用上一個有效交易日的角色與卷宗位置；回合首日沿用回合初始排席。AI 不代為最佳化。
- 休市或全市場資料無效：不增加交易日序號，也不消耗脈絡、風險預算或回合日數；人物生活時間仍可前進。
- 某桌風險煞車降到 `qmax=0`：桌位鎖定，換人或換卷宗不能洗掉。角色仍解讀並更新人物狀態，但不能建立立場，也不能送方向性同伴主張。
- 五人都選擇不做：結果有效，不重抽；仍揭曉第一、第二行動差。
- 五桌都被煞車鎖定：剩餘日為恢復期，可調整人物關係位置與下一回合準備，不能繞過風險結果。

整回合不換位是合法策略。十個有效交易日加速測試中，玩家的中位數必須有 2–5 次因果有效改動；永不換位也要列入盲策略。零至一次代表排席退化成開局設定，每天都換則代表人物沒有連續性，兩者皆為 kill criterion。

## 封席與揭曉

### 08:25：封席

角色只能使用 cutoff 前已進入 allowlist 的證據。每人從自己的四公司卷宗中選擇最多一家、上行／下行／不做，以及 0%、25%、50%、75% 或 100% 的席位信心部位。

「下行」是對稱計分用的合成立場，不宣稱模擬融券、借券、費率、額度或真實放空。產品不用「成交」「下單」「融券」描述這個行動。

封席後，玩家、客服、推薦系統和付費權益都不能提前讀取或改寫決定。

### 日終：揭曉

歷史 episode 依預先封存的官方開盤至收盤資料計算結果。揭曉固定顯示：

1. 最先注意的兩項事實。
2. 思考核心對事實的結構化解讀。
3. 被觸發的一項記憶或人物狀態。
4. 同伴主張及其實際效用增減。
5. 第一、第二合法行動及效用差。
6. 一個已執行的單變量反事實。

只有反事實會改變最後行動時，才能寫「因為這段記憶／這名同伴，他改變了選擇」。若因素只改變效用，文案只能說它強化或削弱原本選擇。

所有可見主張都要帶可展開的 truth badge：

| truth class | 用途 |
| --- | --- |
| `real_fact` | 已封存的市場或公司事實；歷史模式仍遮蔽真實實體與日期 |
| `statistical_sample` | 由真實統計分布約束的合成人口、背景或抽樣 |
| `fictional_setting` | 虛構公司、角色身份、家庭與生活設定 |
| `symbolic_interpretation` | 命盤 motif 與象徵性解讀，不宣稱客觀因果 |
| `simulated_narrative` | 由正式人物狀態與事件投影的模擬敘事 |

同一句不得把不同 truth class 混成無法辨識的事實。Badge 不是免責文字；若互動本身會被讀成買賣建議，仍要改功能。

當期真實模式若未來通過外部 gate，預定 08:25 封席、日終資料完成後向控制者揭曉；實際揭曉時間與公開延遲須由資料 SLA 和逐畫面法律意見共同決定，不能先寫死為 17:00。

## 八回合分區：五席決策分

八名玩家在同一分區累積八個五交易日回合的決策分，不做每週頭對頭，也不做全球總榜。

### Cohort 相對結果

對公司 `i` 的當日開收盤報酬 `rᵢ`，使用同 cohort 其他有效公司的留一法中位數：

```text
bᵢ = median(rⱼ), j 為同 cohort 且 j ≠ i
aᵢ = rᵢ − bᵢ
```

至少需要三家有效同業；不足、停牌、公司無有效開收盤或資料不完整時，該決定作廢：`q=0、DP=0`，不計入參與曝險。不得退回大盤基準。

賽季開始前，以 60 個有效交易日計算公司的 robust scale：

```text
σᵢ = max(
  1.4826 × MAD(aᵢ),
  全 universe robust scale 的第 10 百分位
)
z = 角色方向 × aᵢ / σᵢ
```

上行方向為 +1，下行方向為 −1。結果只看半個歷史波動單位：

- `z ≥ +0.5`，`y=+1`
- `z ≤ −0.5`，`y=−1`
- 其餘 `y=0`

極端值不增加決策分，`y` 永遠只取 −1、0 或 +1。

### 信心校準

令 `q ∈ {0, 0.25, 0.50, 0.75, 1.00}`：

```text
DP = 4 ×（2qy − q²）
E[DP|q] = 8qμ − 4q²
q* = clip(μ, 0, 1)
```

`μ=P(y=+1)−P(y=−1)`，所以 q 表示角色估計的淨方向優勢，不是上漲機率。

| 信心部位 q | 判斷正確 | 判斷錯誤 | 落在雜訊帶 |
| ---: | ---: | ---: | ---: |
| 0% | 0 | 0 | 0 |
| 25% | +1.75 | −2.25 | −0.25 |
| 50% | +3.00 | −5.00 | −1.00 |
| 75% | +3.75 | −8.25 | −2.25 |
| 100% | +4.00 | −12.00 | −4.00 |

沒有穩定優勢時，大信心的期望分數為負。不做是 0 分，可以避開壞決策，不能產生正分。

### 平滑參與曝險

每席整季計算：

```text
Q = Σq
Raw = ΣDP
Coverage = min(1, Q / 8)

若 Raw ≤ 0：SeatScore = Raw
若 Raw > 0：SeatScore = Coverage × Raw
```

八個滿額等價曝險才能保留全部正分；一次 `q=1` 的幸運結果只保留八分之一正分。負分不能靠低參與稀釋，也沒有第八次交易突然解鎖的懸崖。

### 標準化風險煞車

競賽風險報酬固定為：

```text
g = 1% × q × clip(z, −3, +3)
```

每桌追蹤 g 累積曲線的峰谷最大回撤，對照 2%、4%、4%、6%、8% 的桌位線。下一有效交易日 q 上限：

| 回撤線使用率 | 下一日 q 上限 |
| ---: | ---: |
| <25% | 100% |
| 25%–<50% | 75% |
| 50%–<75% | 50% |
| 75%–<100% | 25% |
| ≥100% | 0% |

煞車綁桌位與五交易日回合；換人、換卷宗或換思考核心都不能重置。原始虛擬盈虧可以成為人物故事，但正式競賽與煞車只使用標準化主動結果，兩者不得混成同一數字。

### 分區與反濫用

- 回合分為五席 DP 總和；賽季分為八回合總和。
- 同分先比正分席回合數，再比五席合計標準化最大回撤較低者。
- 精確卷宗、座位、公司、方向、q 與人物因果到第五個有效交易日結算後才對同區與公共世界公開。
- 分區由系統配置，不允許邀請好友進同區。
- 一名驗證自然人每季只能有一支排名隊伍；未驗證只能玩非排名模式。
- 正式賽不提供金錢、可轉售物品或可兌現獎勵。
- 當交易制度或來源發生重大異常，全分區同一 session 作廢。

正式 beta 前使用 500 個按市場 regime 分層、block-bootstrap 的八回合賽季。全上行、全下行、固定 q、永遠不做、最高／最低波動、永遠不換位、產業集中與反向配對等盲策略，必須同時符合：

- 平均賽季分的 95% 信賴區間上界 ≤ 0。
- 進前二率的 95% Wilson 上界 ≤ 30%。
- 任一市場 regime 四分位的進前二率上界 ≤ 35%。

未通過時，賽制不得進 beta。

## 角色引擎

模型是受限的市場解讀器，不是最終決策者。正式狀態、行動和關係更新由 deterministic simulator 計算。

### 最小正式狀態

```text
不可變身份與 seed
四軸人格：探索、證據取向、支配、損失敏感
命盤 motifs：控制、歸屬、新奇、衝突、退避、認可
產業與資訊技能
疲勞、家庭壓力、健康負荷、掌控感
情緒：正負向、喚醒、掌控
最多 12 段帶觸發、結果、顯著與衰減的記憶
關係：信任、服從、義務、怨懟
每家公司的上行／下行支持、不確定度、新鮮度與矛盾證據
自我校準：信心斜率、信心偏差、近期高 q 錯誤與低 q 正確
動態需要：安全、控制、歸屬、認可
桌位、卷宗、脈絡日數、同伴來源、風險煞車
```

完整虛構出生資料會轉成有上限的命盤 motif，只影響注意、意義與行動門檻。血型只形成角色自我敘事和他人刻板預期，不改智力、證據品質或客觀勝率。

### 思考核心的權限

每個核心只收到 cutoff、候選公司、action mask、evidence allowlist 與輸出 schema。它不取得日後結果、對手決定、自由網路、任意公司或未列入 allowlist 的背景常識。

核心對卷宗內每項 allowlisted evidence 分別輸出關聯、方向效果、可靠度與不確定度。它不判斷跨事實矛盾；版本化的 deterministic normalizer 讀完整 manifest 後建立 contradiction graph。只有 simulator 選中的三個注意槽能進入行動效用。核心不能輸出最終公司、方向、信心、價格預測、自由長文或人物狀態修改。

思考核心可有不同解讀風格和盲點，但對所有玩家同時可用，不按勝率販售。歷史保存當時原始結構化輸出，不因模型升級重算。

### 一次社交交換

每人獨立完成初稿，再同時送出一份密封主張：公司或產業、立場帶、信心帶、最多兩個證據 ID，以及支持／挑戰／警告意圖。

- 每人只收左手邊一份。
- 沒有第二輪或群聊。
- 社交因素最多占最終行動效用 25%。
- 信任提高取信權重；怨懟提高反駁權重。
- 若主張與卷宗沒有公司或產業交集，只能改不確定度。
- 接受或拒絕由正式狀態計算，不再呼叫模型。

### 最終行動效用

對每個合法公司 c、方向 d 與 q：

```text
E(c,d) = 注意槽中的 evidence appraisal × 來源可靠度 × 資訊技能 × 方向
B(c,d) = 既有信念帳本
M(c,d) = 觸發記憶偏置 × 顯著度 × 衰退 × 記憶信心
S(c,d) = 同伴主張 × 信任／服從／怨懟，絕對貢獻上限 25%
C(c,d) = clip(E + B + M + S, −1, +1)

R = 1 + 損失敏感 + 不確定度 + 疲勞 + 風險煞車壓力 − 桌位熟練
U(c,d,q) = 2qC(c,d) − q²R
U(NO_ACTION) = 恢復價值 + 過載避免價值
```

只在 `q≤qmax` 的行動中取最高效用。`不做` 永遠是正式候選。探索性改注意新奇事實，證據取向改 reliability 權重，支配／協作改社交取信，損失敏感改 R；命盤 motif 只能經動態需要與事件觸發作用，不能直接替上行或下行加分。

### 結果後更新

每次 settled session 依固定順序更新：

1. 以結果和 q 計算 surprise。
2. 高 q 錯誤降低信心斜率；低 q 正確只小幅提高。
3. 結果相對預期更新情緒；社交衝突更新喚醒；是否遵循自己判斷更新掌控感。
4. 依 surprise、需要、關係衝突與顯著度寫入或合併最多 12 段記憶。
5. 依來源誠實更新 trust，依同伴主張後續有效性更新 deference；不能只因正分就增加所有關係。
6. 未解假說衰退，新證據進入信念；換卷宗只降低可取用性，不刪除信念。
7. 依注意負荷、q、衝突、休息與桌位適配更新疲勞及需要。
8. 同角色、同桌、同卷宗完成有效交易日後更新脈絡。

所有係數與順序帶 policy version，賽季中凍結。

### 重播與可證偽

- **法證重播：** 使用原始狀態、證據、模型輸出、政策與 seed，必須 100% 重現行動與狀態。
- **單變量反事實：** 固定模型解讀，只改一項記憶、情緒、桌位、同伴或命盤 motif，重新計算行動。
- **核心替代推演：** 使用另一思考核心重新解讀，只能標為替代情境，不能改寫歷史或當成精確因果。

賽季指派前發布 `SHA256(season_secret)` commitment；需要平手、公共生活或敘事變體時，以 `season_secret + episode + character + session + phase + policy` 產生 HMAC seed。seed 不用來製造人格差異。第一份符合 schema 的 appraisal 永久保存；無效輸出走 deterministic fallback，不得重抽到較好看的結果。賽季後公開 secret 供驗證。

單變量反事實固定原始 appraisal、其他人物狀態、seed 與 policy，只介入一項同伴、記憶、情緒、桌位、命盤 motif 或脈絡事實：

- 最終 action 改變：可稱「決定性因素之一」。
- action 不變，但第一／第二效用差改變至少 0.10：可稱「強化／削弱決定」。
- 未達門檻：不得進主因果帶。

驗收只計算因素具備有效 exposure 的決策：各人格／命盤／記憶／情緒／關係因素，效用差改變至少 0.10 的比例須為 15%–60%，action flip 須為 3%–20%；任一單因素不得解釋超過全體 flips 的 25%。相同 state＋evidence＋appraisal 必須 100% 得到相同行動；不同角色在相同證據下的行動差異率為 15%–45%；純敘事改寫造成的 action 差異率必須是 0。

另設無 evidence 對照：排除中性結果後，方向正確率須落在 45%–55%，平均 DP 不得為正，否則視為歷史結果洩漏。至少 70% 測試者要能把角色差異歸因於可描述的持續特質，而非隨機台詞。

## 角色的一生

每位訂閱者建立或領養自己的五名唯一角色。人物有一條連續人生，不可交易、邀請、拍賣或被他人接管。

```text
建立完成 → 我的五席 → 公開自主生活 → 回到我的五席
                     ↘ 季末退休典藏
```

- 訂閱有效：玩家擁有排席權、完整歷史與無廣告體驗。
- 訂閱失效：在下一個回合邊界失去排席權；五人保留人生，在公開世界依統一規則自主生活。
- 公開自主：任何人都只能延遲觀看，不能控制、贖回或競標。
- 恢復訂閱：下一個回合邊界取回同一五人的排席權，不回檔、不重生；正式分區於下一季加入。
- 退休：只在季界由玩家主動決定，與付款無關。

記憶分為世界公開、與原玩家共享、角色私有三層。重新取得排席權不等於讀取角色的全部內心。失訂本身不扣信任、不製造被拋棄記憶，也不提高受傷或退休機率。

## 首屏

首屏是五席圓桌：

```text
┌─────────────────────────────────────────────────────────────┐
│ 盤勢・眾生       距離封席 00:41:23       Episode 03／40       │
├──────────────────┬──────────────────────┬───────────────────┤
│ 五個人            │ 五張桌位與五份卷宗      │ 昨日關鍵因果         │
│ 狀態／脈絡深度     │ 拖曳排席／合法行為預覽   │ 決定→結果→反事實      │
│ 本回合風險煞車      │ [確認封席]              │ [進入完整回放]        │
├──────────────────┴──────────────────────┴───────────────────┤
│ 分區決策分｜角色人生｜卷宗證據｜生涯典藏                     │
└─────────────────────────────────────────────────────────────┘
```

首頁十秒內要回答：現在能否排席、誰有脈絡、哪席受風險煞車、昨日哪個因素真的改變了行動。人物生活、完整證據與分區位於第二層。

視覺採墨黑、紙色、收斂的銅與一個冷色資料訊號。命盤在封席與反事實時形成克制的動態紋理；分數、資料狀態與風險保持高對比、可讀並提供 reduced-motion 等價畫面。禁止賭場閃光、霓虹交易終端、裝飾性粒子與假即時跳價。

## 上市分層與法律 gate

本文件不是法律意見。產品預設採最保守可驗證路徑：

| 模式 | 產品決定 |
| --- | --- |
| 歷史 beta | 可做隔離版：至少一年前資料、虛構公司名、遮蔽日期；無金流、廣告、當期新聞、ticker 搜尋、交易清單匯出 |
| 當期封測 | 不對外開放；完整 prototype、資料權利與逐畫面法律問題取得書面意見前，只能內部受控 dogfood |
| 公開正式版 | 預設歷史封存或虛構公司；當期真實公司必須經書面法律意見、資料授權及核准營運路徑後，作為獨立模式啟用 |

主管機關將直接或間接對個別有價證券提供分析意見或推介建議列入證券投資顧問定義；一般理財網站的界線也排除個別證券分析。角色若輸出「真實公司＋上行／下行＋信心部位＋依據」，延遲揭露不會自動將它變成一般資訊。交易資料能在官方網站讀取，也不代表具有模型輸入、衍生計分、保存與商業再散布權。

歷史 beta 不顯示真實公司、代號、日期、角色命中率、最強選股角色、分享部位、外部行情連結或預測式招募文案。是否能在 episode 結束後揭示真實對照，仍須外部法律與資料權利確認。

每個歷史 episode 上線前都要有 rights manifest，逐項涵蓋開收盤、產業分類、指數、公司事件、公告／新聞文字與圖片；公開互動、模型輸入、衍生 cohort／DP／反事實、保存／備份／跨境、回放／截圖／直播／UGC，以及授權終止後的留存都要有明示權利。未明示允許 episode 後揭露真實公司或日期時，映射永久不公開。

歷史隔離與當期真實是不同內容域，不是同一功能的 boolean flag：

| 邊界 | 歷史隔離模式 | 當期真實模式 |
| --- | --- | --- |
| Domain | 只有虛構公司 ID、episode 相對日與 cohort；真實映射不進玩家域 | 真實公司是另一實體，須帶司法管轄、授權與審核版本 |
| API | 不接受 ticker、真實名稱／日期或反向查詢；永不回傳映射 | 預設不存在公開能力；完整 gate 後才有獨立契約 |
| Cache | 所有內容帶 mode 與 rights scope，不得命中真實內容 | 獨立權利期限、撤銷與清除，不進歷史公共快取 |
| Search | 只搜虛構公司、角色與 episode，不能以事件反查映射 | 獨立索引且預設關閉 |
| Notification | 只發人物、回合、封席與虛構事件，不含真實公司與方向 CTA | 獨立逐句審核 catalog；法律 gate 前整項停用 |
| Analytics | 一般分析只收 opaque ID；BI、廣告、A/B 與推薦拿不到映射 | 公司、方向與人物決定不進一般成長分析或廣告受眾 |

`mode` 在 episode 建立後不可轉換。使用者、客戶端與一般營運人員無權啟用當期模式；啟用條件必須引用指定法律意見、授權契約、UX 審查與誤讀測試版本。模擬規則可共用，內容識別、公開政策、搜尋、通知與分析權限不可共用預設值。

正式版若進入當期模式，必須逐畫面審查訂閱頁、商店頁、通知、角色決定、分區榜、搜尋與分享卡；不得提供全服多空熱度、當日角色共識、依角色決定排序的公司頁、券商／開戶／下單連結、目標價或未來機率。至少 30 名目標使用者完成訊號誤讀測試；超過 10% 表示會把內容用於隔日真實交易時，該模式不得公開。

## 商業模式

封測不串金流、不顯示廣告。正式歷史版預定：

| 免費觀看 | 訂閱排席 |
| --- | --- |
| 延遲觀看已完成的公共人物與精選因果 | 排自己的唯一五人 |
| 可看分區結果 | 看完整生涯、因果與反事實 |
| 廣告只出現在自然段落 | 無廣告 |

付費購買的是排席權、歷史深度與清爽體驗；不提供較準模型、較早資料、角色贖回或績效優勢。廣告不得插在封席倒數或人物脆弱時刻，也不得用金融焦慮促成訂閱。

## 動工前必須證明

1. 每日排席在 60 秒、十次動作內完成，且不像填表。
2. 第三日後，至少 80% 測試者能說出一條排席如何改變角色決定。
3. 每次揭曉都有第一／第二行動、效用差與一個單變量反事實。
4. 法證重播一致率 100%；敘事修改不會改變正式狀態。
5. 角色差異率、各因素影響率與社交上限落在憲法區間。
6. 500 個分層 bootstrap 賽季通過平均分、30% 全局與 35% regime 的盲策略上界。
7. 免費觀看者不把內容解讀成當下買賣建議。
8. 失訂與恢復不被理解成角色綁架或付費贖回。

未驗證前，只能做歷史資料契約、決策與計分模擬、可點擊排席原型、角色反事實實驗與測試骨架。當期真實公司、城市、公地社交、開放聊天、模型商店、全球榜與真錢獎勵都不屬於首版架構前提。
