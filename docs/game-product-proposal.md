# 《盤勢・眾生》產品企劃書 v0.6

_2026-07-20｜方向重置版｜本文件是產品規則的唯一依據。技術、FigJam、Figma、文案與程式如有衝突，先改本文件。_

## 0. 這次重切了什麼

前一版把人做得很細，卻把市場當成了人物故事的佈景。結果會是好看的角色檔案、沉重的週卷，以及沒有必要每天回來的使用者。

新版不再把《盤勢・眾生》定義為人物觀察劇或歷史教材。

> **盤勢是一座與真實市場時間同步運行的 AI 城市。你培養五種思考方法，看它們在同一個未知裡分化成不同命運。**

產品的基本回路必須永遠成立：

```text
使用者引導 → 角色感知與行動 → 真實世界向前一天 → 後果揭封 → 關係／記憶／方法改變
```

任何功能若沒有讓這條回路更清楚，就不屬於首版。

## 1. 產品不是什麼

- 不是券商、實盤、跟單、AI 選股或績效競技場。
- 不是用角色包裝的聊天機器人。
- 不是只有回放的歷史盲測課程。
- 不是讓使用者遙控小人買哪一檔、買多少的養成遊戲。
- 不是靠抽卡、戰力、模型等級或角色依戀來付費的內容農場。

角色會進行虛擬交易，因為沒有承擔後果的行動，就沒有未知；但交易只屬於世界模擬，不能變成對使用者的交易指令。

## 2. 一句定位與核心幻想

**定位：**

> 真實市場每天往前走；你養的不是投資答案，而是五種面對未知的方法。

**核心幻想：**

使用者擁有一間觀測所，能把不同性格、記憶、關係與思考核心的居民送入同一座市場城市。不是替他們下注，而是決定他們這週優先追問什麼、先看誰、如何接觸反證、要不要坐下來談。下一個交易日到來後，使用者看見同一個世界如何把五種方法推向不同的行動、關係與後果。

**固定權限句：**

> 你能塑造他怎麼面對未知，不能替他向市場下注。

## 3. 三個不可替代的時刻

### 收盤鐘後：同一事實，五條路

世界時鐘走到一份市場事實可被使用的時刻。盤勢帶來今日五個值得討論的公司事件；居民在研究室、茶館與街區接到不同片段。同一份公告，有人先找反證，有人盯住風險，有人被朋友的表態推著走。使用者先看到分歧，再決定要把有限引導用在哪裡。

### 隔日揭封：後果有來路

上一日封存的紙上決策在下一個市場節點執行，新的市場事實再把後果帶回來。畫面不宣告誰神準，而是可回放「他當時看見什麼、漏掉什麼、哪段記憶被勾起、誰改變了他」。

### 公共城市：你離開後，世界沒有停

當使用者不再引導角色，已同意公開的居民回到公共城市繼續生活、研究、結識與改變。別人可以追更他們的公開軌跡；使用者的私人筆記和未揭示的引導永遠不公開。這不是懲罰或情緒勒索，而是世界持續存在的證明。

## 4. 時間、未知與紙上交易

### 世界時鐘

- 每個世界 tick 只讀一份已封存的 fact manifest。
- 角色只可看見 `available_at <= logical_at` 的事實；來源發布、實際取得與產品可用時間必須分開保存。
- 上游修正永遠追加 revision；不重寫角色當時看見的世界。
- 缺資料、延遲、授權失效或 manifest 驗證失敗時，世界停住並直接顯示最後可信時間。不能用模型補劇情。

### 虛擬決策的時間契約

為了保留真正未知、又不把角色當即時訊號，首版採固定的 D／D+1／D+2 節奏：

| 時刻 | 世界發生的事 | 使用者看見什麼 |
|---|---|---|
| D 收盤後 | 讀取 D 的封存事實；角色研究、對話並封存紙上決策意圖 | 推理、矛盾、未解問題與引導造成的變化；股票、數量與方向仍封存 |
| D+1 收盤後 | 引擎以預先定義的虛擬成交規則執行 D 的意圖 | 角色已採取行動，但公開詳細部位仍延遲 |
| D+2 收盤後 | 新事實到來；帳本標記、關係與信念更新 | 完整回放：行動、理由、資料、後果與後續修正 |

私人觀測所可比公地早一個節點看到角色的「意圖類型」，但不顯示可以被抄作即時操作的標的、數量或進出價格。公地一律在 D+2 才取得完整回放。沒有全服報酬排行、模型排名、勝率、目標價或買賣提醒。

紙上帳本是世界狀態，不是成效宣傳。它要正確處理現金、持倉、分割、除權息、停牌、無成交與資料修訂；資料不完整即不結算。

## 5. 使用者能改變什麼

### 五席

五席是使用者每週可引導的五位居民，不是每週一次性生產五個新角色。角色存在長期名冊與公共城市；被放進五席時才取得完整感知、互動與高頻模擬。封測不扣費，但必須使用正式 entitlement 狀態 `beta_full_access`。

### 每日三枚引導籤

每個交易日，使用者最多投入三枚引導籤；可不用，也不能囤積。每枚只能對一位五席居民生效，並會留下可回放的因果痕跡：

| 引導籤 | 改變的東西 | 不能改變的東西 |
|---|---|---|
| 問一問 | 角色本日必須先列出未知與反證門檻 | 交易方向與結論 |
| 遞線索 | 在已允許的資料包中提高某個證據群的注意順位 | 角色可看見的資料邊界 |
| 約一席 | 安排兩位角色於下一個社交節點交換各自已看見的內容 | 對方必須同意、必須改變想法或交易 |

每週開始時，使用者為每一席設定一條研究命題，例如「在結論前找出兩個會推翻自己的人」或「把治理與現金流分開看」。命題不是個股、方向或績效目標，卻會長期改變注意、關係與記憶。

### 使用者永遠不能做的事

- 指定股票、買賣、數量、價格、槓桿、停損、出場或表現目標。
- 修改角色的情緒、偏誤、關係、記憶、帳本或市場資料。
- 付費提前揭封、重跑已知結果、買更高勝率或取得更快資料。
- 用私人筆記訓練模型、生成公共內容或對外揭露其他使用者。

## 6. 居民不是貼標籤的角色卡

每位居民都是完全虛構、年滿 18 歲的角色。真實世界只提供彙總人口分布、世代背景與公開事件，不能匯入、模仿或拼貼真人。

角色資料分五層：

1. **出生底盤**：年齡、地區、職涯、生活責任、四軸性格偏好、血型、虛構出生資料與人盤。
2. **心理參數**：損失厭惡、確認偏誤、模糊厭惡、從眾、權威依賴、自尊脆弱與核心需求。
3. **動態狀態**：壓力、疲勞、信心、羞恥、FOMO、孤獨、信任與當日注意力。
4. **結構化人生**：家庭金錢觀、成功與失敗、未竟關係、人生謊言與會被什麼喚起的記憶。
5. **關係網**：信任、欣賞、依賴、競爭、嫉妒、權力差與未說出口的期待。

命盤、四軸與血型不會決定股價、打開未來或提高績效。它們透過下列路徑進入世界：

```text
人格／命盤／人生 → 注意什麼 → 如何解讀 → 對誰有反應 → 做什麼 → 如何面對後果
```

血型只作極低權重的敘事風味。四軸偏好是連續值，介面不用正式 MBTI 商標或人格診斷語言。

## 7. 思考核心：差異是失誤型態，不是戰力

使用者可為五席選擇版本化的 **思考核心**。它不是「智商」分數、模型商店或付費優勢；公開介面不顯示供應商與基礎設施。預設核心為「平衡・26B」，後端以私有 capability alias 對應實際容量與版本。

封測同時提供三種同權核心：

| 思考核心 | 擅長 | 容易犯的錯 |
|---|---|---|
| 平衡・26B（預設） | 整理互相衝突的證據與理由 | 把衝突磨得太平，延後表態 |
| 反證 | 主動尋找推翻自己的資料 | 過度等待，錯過可行但不完美的行動 |
| 探索 | 建立跨事件、人物與敘事的連結 | 容易被近期訊號與故事牽引 |

所有核心用相同事實邊界、紙上帳本、交易規則與引導權。模型版本、提示、輸入證據包、規則版本、輸出與政策檢查均要留存，讓一段世界歷程可重播。模型不選擇任意資料、不改寫帳本、不決定是否發布，也不直接生成世界真相。

## 8. 五種資料身分

每個可見 claim 都必須附 `truth_class`，而且一個句子若混了兩種身分，就拆成兩個可展開區塊：

| 身分 | 例子 | UI 的固定讀法 |
|---|---|---|
| 真實資料 | 公司公告、交易日、價格、停牌 | 真實資料／來源／時間／修訂 |
| 統計取樣 | 收入帶、職業分布、世代條件 | 統計取樣／不代表真人 |
| 虛構設定 | 角色父親的印刷廠、私人記憶 | 虛構設定／角色人生 |
| 象徵解讀 | 控制、責任、被看見的命盤主題 | 象徵解讀／不代表市場方向 |
| 模擬敘事 | 他延後表態、去找朋友談 | 模擬敘事／由角色狀態生成 |

這些標籤必須出現在世界事件、人物內心、回放、影片、分享卡與空／錯誤狀態，不只在資料來源頁。

## 9. 一日、七日、八週循環

### 一日：昨天的引導，今天有後果

```text
收盤鐘響 → 讀取封存事實 → 同證異讀與社交傳遞
→ 使用者投引導籤 → 角色封存意圖 → 次日虛擬執行
→ 再隔日完整揭封與 replay
```

使用者每天只需花 30 秒看一個分岔；投入五分鐘時可使用引導籤、比較兩位角色的感知包與閱讀證據。沒有簽到懲罰。

### 七日：方法會不會改變人

使用者安排下週五席、重寫研究命題、保留或調整思考核心。世界會結算關係張力、反覆偏誤與記憶改寫，產出一份「分岔報告」：哪一段關係改變注意、誰更願意改口、誰把同一錯誤說成了新理由。

### 八週：不是誰賺得多，而是誰活成了什麼方法

八週構成一季。使用者可重播關鍵分岔，觀看五條方法軌跡如何在不同市場環境中變得穩定、僵化、分裂或成熟。紙上帳本可查看，但不產生排行榜、戰力或全球冠軍。

## 10. 首屏與資訊架構

示範圖可保留墨黑、紙纖維、黃銅、礦物藍與細緻角色資產；但不可再用角色履歷、雷達圖與帳本塞滿第一眼。首頁必須先讓人看見世界正在別處發生。

```text
┌──────────────────────────────────────────────────────────────────┐
│ 盤勢・眾生  世界時鐘：收盤後 01:42  可用事實：18／18              │
├─────────────┬────────────────────────────┬───────────────────────┤
│ 我的觀測所   │        活的市場城市          │ 今日的分歧             │
│ 五席與引導籤 │  街區／研究室／關係線／五事件  │ 同一事實的不同路徑      │
│ 角色此刻狀態 │  點一人，進入他的所在與時間線  │ 未知封條與下一次揭封     │
├─────────────┴────────────────────────────┴───────────────────────┤
│ 世界脈衝：研究 → 對話 → 封存意圖 → 虛擬執行 → 後果 → 記憶         │
└──────────────────────────────────────────────────────────────────┘
```

第一層是世界、事件與分岔；第二層才是角色、關係、命盤、帳本、記憶與全證據回放。公司命盤研究從事件卡的「象／證／界」自然展開，不把世界做成醜陋的金融 dashboard，也不把資料藏進神祕文案。

## 11. 公地、權益與封測

### 未來商業化

| 權益 | 免費觀看者 | 盤勢 Pro |
|---|---|---|
| 公共城市與延遲回放 | 可看、可追蹤有限居民 | 完整觀看、完整歷史 |
| 建立與引導 | 無 | 每週五席、研究命題、每日引導籤與私有觀測所 |
| 思考核心 | 觀看公開核心 | 五席可選全部同權核心 |
| 私人筆記與匯出 | 不建立 | 完整、永不公開 |
| 廣告 | 依盤勢整體廣告政策，只在自然斷點 | 無廣告 |

訂閱購買的是引導權、私有工作流與無廣告，不是更快市場資料、更多資訊、勝率、報酬或角色忠誠。

### 角色離開引導權後

建立角色時，使用者必須明白選擇其預設歸宿。產品預設為 **公共城市**，因為這是世界的核心規則；可選擇「私有封存」取代公開。失去權益時，當週走完後才生效；公共城市只投影已審核的公開事件、延遲帳本與關係，永不投影私人筆記或使用者未公開引導。重新取得引導權可把角色放回空席；不販售贖回、不做倒數、哭泣或「他在等你」文案。

### 封測

封測所有受邀帳號都有 `beta_full_access`。不串金流、不顯示廣告、不設商店、不顯示試用倒數；用正式 entitlement、public-consent、guidance-lost 與 recall 狀態機驗證未來流程。

## 12. Production-shaped beta：縮世界，不縮契約

封測不是傳統 MVP；它保留終局的 manifest、感知、紙上帳本、關係、模型版本、公共投影、replay、身份與 API 契約，只限制可同時啟用的世界範圍：

| 範圍 | Beta 決策 |
|---|---|
| 城市 | 一座「臺北觀象城」，有研究室、公告牆、夜市茶館與公共廣場四種場景 |
| 居民 | 50 人：10 位高頻居民、40 位規則式低頻居民；可容納封測使用者的五席投影，但只有被關注者升級為高頻敘事 |
| 公司事件 | 8 類：重大訊息、月營收、財報、法說、停復牌、除權息／分割、量價異動、盤勢研究事件 |
| 思考核心 | 3 種同權核心，包含預設平衡・26B |
| 盤勢內容 | 每日五個研究事件；每個事件都有象、證、界與反例／資料不足狀態 |
| 觀察週期 | 真實交易日 tick；只有資料授權、延遲與訊號誤讀 gate 通過後才開啟當期世界 tick，未通過即暫停，不用歷史模式冒充 |

驗證的不是「大家喜不喜歡角色」，而是：使用者會不會因為昨天引導造成的可讀分岔，在下一個交易日回來。

## 13. 終局世界引擎

```text
sealed fact manifest
  → world clock / visibility
  → perception packet（看見、漏掉、轉述）
  → trait + memory + relationship + thinking core
  → deterministic paper-order intent
  → D+1 execution / D+2 consequence
  → ledger + state transition
  → constrained narrative + truth badges
  → private view / delayed public projection / replay
```

**必須規則化：** 時間、授權、資料可見性、感知權限、虛擬成交、帳本、關係狀態、權益、公開投影、資料身分、審計與 replay。

**可由模型生成：** 在既定證據與結構化狀態中產生角色措辭、內心戲、對話、場景、週卷與分享內容。模型輸出若引用不存在的事實、捏造數字、變成投資指令或不符 truth class，事件必須 held-for-review，不得發布。

**擴張原則：** 未被關注的居民按低頻規則推進；被放進五席、被追蹤、位於重大關係轉折或需要公開內容的居民才建立完整感知包與敘事。這讓世界能從 50 人增加到數十萬居民，而非隨人數線性生成所有對話。

## 14. 五件絕對不要做的事

1. 開放無限自由聊天，讓角色一致性、成本與可回放因果失控。
2. 顯示即時或可複製的個股操作、模型排名、勝率或「今天誰最準」。
3. 將思考核心做成戰力、盲盒、付費更聰明或更快資訊。
4. 用角色孤單、失去關係或即將消失逼人續訂。
5. 對每個居民、每個 tick 都生成長篇內容，讓世界變成昂貴且無因果的 AI 內容農場。

## 15. 產品方向已凍結的決策

這次不留下模糊選項；以下三項是 v0.6 的固定決策：

1. **最小玩家行動：** 每位使用者每天最多三枚引導籤，僅能改變問題、注意力或社交，不可改交易。
2. **揭示節奏：** D 收盤封存意圖、D+1 虛擬執行、D+2 公開完整回放；私人觀測所只早一個節點得到不含標的與價格的意圖類型。
3. **思考核心：** 封測有三個同權、以失誤型態區分的核心；預設為平衡・26B，供應商映射只在私有運維設定中存在。

仍需在寫入當期市場世界前通過資料授權、延遲規則、金融風險審查與訊號誤讀測試；這些是開關條件，不能成為把虛擬交易或真實未知從產品核心刪掉的理由。

## 16. Pre-code P0：不改方向、但必須先關上的五道門

最終架構 gate 的結論是可進入 pre-code P0，而不是可直接散開寫畫面與 worker。以下是實作前的共同交付物：

1. **世界事件目錄：** 將引導、token、意圖、執行、帳本與回放定義成版本化 event；client 只能寫引導命令。
2. **世界鐘：** 固定 D／D+1／D+2、交易日曆、假日、停牌、補件、更正和 fail-closed saga；同日不可有部分居民先前進。
3. **紙上世界物理：** 凍結 cash、曝險、成交、未成交、公司行動、修約與四捨五入規則，並以 golden fixtures 重播。
4. **可見性邊界：** 用逐欄 policy 規定 private／public 在各個時間點可讀的資料；兩者的資料查詢和快取從源頭隔離。
5. **高低頻居民：** 凍結升降級、更新頻率、關係傳播和 fallback；50、5,000、50,000 居民只能靠加算力和讀模型擴張，不改世界事件形狀。

這五道門的完整技術契約以[技術架構 §7](./game-technical-architecture.md)為準。它們不會讓產品退回傳統 MVP；它們確保首版就是可擴張的活世界。

## 17. 成功條件

封測要證明的是世界回訪，而非報酬：

- 使用者能在一週後清楚說出至少三位居民的「方法與盲點」，不是只記得外型。
- 使用者能指出自己的一枚引導籤改變了哪個感知、對話或後續行動。
- D+1 與 D+2 的分岔回放有穩定回訪，而非只在造人當日熱鬧。
- 使用者把角色行為誤認成當期投資訊號的比例低於產品預先設定的安全閾值。
- 任何可發布內容中，虛構、象徵與真實資料的身分標示缺漏率為 0；模型捏造事實、數字或指令的發布率為 0。

如果玩家只把它當作選股訊號、只想直接下單、或在兩輪調整後仍無法感受到引導與後果的連結，就停止擴建城市，先修正核心 loop。
