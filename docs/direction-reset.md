# 《盤勢・眾生》方向重置

_2026-07-20｜已吸收獨立 Pro 審查；方向已寫入正式企劃、技術架構、backlog 與 FigJam。_

## 判決

舊稿把核心縮成「五個角色的可稽核觀察劇」。這很安全，卻不是創辦人要做的東西：它把交易、未知、市場時間與眾生互相影響都降成了背景，使用者每天只是在讀整理得很好的角色檔案。

新的產品不是讓人替 AI 下單，也不是把人物包在紙上交易器裡。

> **盤勢是一座由真實市場時間推動的 AI 眾生沙盒。你不替角色下注；你培養五種面對未知的方法，看它們如何在同一個世界裡活下去。**

它要讓人回來的不是「這個模型猜對了沒有」，而是三個同時發生的問題：這個人今天看見了什麼？他被誰改變了？明天揭封後，他還會相信自己嗎？

## 四個方向翻轉

| 舊稿 | 新方向 |
|---|---|
| 角色檔案與週卷是主產品 | 可俯瞰、可進入、持續推進的市場世界是主產品；檔案是深入世界的入口。 |
| 歷史盲播是首要核心 | 真實交易日的未知是核心；歷史回放只用於新手、測試與重播。 |
| 玩家主要遞反證、看自省 | 玩家每週配置五席，塑造研究命題、注意力、資料路徑、社交安排與思考核心。 |
| 淡化紙上交易以避開訊號感 | 保留虛擬交易與後果，但用資料可用時間、延遲揭示、無實盤與無排行處理誤讀。 |

## 世界的基本承諾

### 1. 時間是真的

世界只在已封存、已允許使用的事實出現後向前走。角色在交易日 D 看見的資料，不能包含 D+1 才會知道的內容。D 的研究會封存虛擬決策意圖；D+1 依固定規則執行；D+2 的新事實才揭封後果。

因此市場不是一段已寫好的劇本，也不是模型假裝不知道答案的歷史影片。

### 2. 人是真的虛構的

居民都是完全虛構的成年人。人口統計只約束合理分布，真實年代與公開事件只提供背景，任何個人遭遇、關係、生日與記憶都不可回連真人。

每個角色的四軸性格偏好、命盤、血型、年齡、工作壓力、金錢記憶、關係與情緒，都會改變他看見什麼、相信誰、何時逃避與如何承擔結果。它們不會改變價格、打開隱藏資料或提高勝率。

### 3. 引導是真的，但指令不是

使用者擁有的不是遙控器，而是一間觀測所：

- 為角色設定這週的研究命題與注意力方向。
- 在可見資料中指定優先研究的來源或反證規則。
- 安排兩個角色交換資訊、共事或暫時保持距離。
- 為角色選擇版本化的思考核心。
- 留下私人的觀察與假設。

使用者永遠不能指定個股、買賣、價格、數量、槓桿、停損或出場。角色仍會做虛擬交易，因為不做決定就沒有未知的後果；只是行動必須出自角色在當時的世界，而非使用者的指令。

### 4. 每句話都知道自己是什麼

世界中的可見內容都附帶一種身分：

- 真實資料：市場、公司公告、時間與天氣。
- 統計取樣：角色的合理人口條件。
- 虛構設定：人物生平、記憶與關係。
- 象徵解讀：命盤所觸發的主題。
- 模擬敘事：系統根據狀態寫出的反應。

這不是資料來源頁的附註，而是世界的閱讀規則。

## 三個不可替代的時刻

### 收盤鐘後：同盤異命

世界時鐘走到可用事實的邊界。市場地圖亮起五個今日值得討論的區域；同一份公司公告在不同街區被讀成擴張、警訊、機會或暫不判斷。使用者先看見分歧，再決定要走進誰的房間。

### 明天揭封：後果不是答案

下一個市場節點到來，角色的虛擬帳本、關係與信念同時更新。畫面不宣布誰「神準」，而是讓使用者看見：他原本看見什麼、漏掉什麼、誰影響了他，以及新的事實究竟推翻了哪一段理由。

### 公地：離開後世界仍然會走

未被引導的角色會在公共世界中繼續研究、工作、結識與改變。使用者從私人的五席走到公共街區，看見其他人的居民與其回放脈絡；公共性來自世界活著，不來自績效排行。

## 首屏應長什麼樣

示範圖的墨、銅、紙、細緻角色與密度可保留，但資訊階級應重排：

```text
┌────────────────────────────────────────────────────────────────────┐
│ 盤勢・眾生   世界時鐘：收盤後 01:42   今天可用的事實：18／18        │
├─────────────┬──────────────────────────────┬───────────────────────┤
│ 我的觀測所   │         活的市場世界          │ 今日的分歧             │
│ 五席／本週   │  互動城市／研究室／街區         │ 同一事實的 3 種讀法     │
│ 角色狀態     │  五個受注目的公司事件在此發生    │ 關係牽引／未知封條      │
│ 引導動作     │  點角色進入他的時間線與所在場景  │ 進入一位角色             │
├─────────────┴──────────────────────────────┴───────────────────────┤
│ 世界脈衝：研究 → 對話 → 虛擬決策（明日執行） → 後果揭封 → 記憶變化 │
└────────────────────────────────────────────────────────────────────┘
```

角色履歷、內心戲、關係圖、命盤與帳本都在第二層；首屏先讓人感到「事情正在別處發生」。價格與帳本只用清楚、克制的資料標記，不佔據情緒敘事的主導地位。

## 思考核心不是智商商店

「智商」在產品內統一稱為 **思考核心**。它是一個可重播的版本化組合，而不是能力分數或可付費購買的戰力。

每個核心公開其研究風格、可處理的資料型態、常見盲點與版本紀錄。例如預設的 26B 平衡核心可長於整理矛盾證據、短於極少樣本下的結論；另一核心可以更擅長關係對話、但更容易被近期事件帶動。所有核心都有相同資料邊界與虛擬交易規則。

模型不產生世界事實、不選任意證據、不直接寫帳本。規則引擎先固定感知、狀態與允許行動，模型只在該邊界內提出或呈現解釋；版本、輸入、輸出、政策與 replay 都必須留痕。

## 五席與公共世界

五席是使用者每週可引導的角色位置，不是每週一次性產生五個角色。每位使用者可保有長期名冊；被選入五席的人會獲得完整世界更新、引導、互動與私有觀察。

未來訂閱失效後，使用者失去引導權。建立角色時須明確選擇其預設歸宿：進入公共世界或轉成私有封存；若選前者，角色與已審核的公共軌跡會繼續存在，私密筆記、未公開的使用者選擇與敏感投影永遠不外流。封測先模擬這個狀態機，不收費、不插廣告。

## 最小但完整的世界引擎

```text
sealed market fact
  → 世界時鐘與可見性
  → 角色感知（看見／漏掉／由誰傳來）
  → 記憶、關係、情緒與思考核心
  → 規則式虛擬決策與下一日執行
  → 紙上帳本與市場後果
  → 可驗證敘事、公共投影與重播
```

規則化：時間、資料可見性、感知權限、帳本、交易執行、關係狀態、權益、公開投影、資料標籤與 replay。

模型生成：在既定結構內的對話、不同人物的解讀措辭、場景、週卷與分享片段。若模型失敗或違反資料／語言邊界，世界可保留規則結果但不發布敘事。

## 已凍結的三個產品決策

1. **角色歸宿**：建立時明確選擇歸宿；預設進公共城市，私有封存是可選替代。私人筆記永不公開。
2. **揭示節奏**：D 封存意圖、D+1 虛擬執行、D+2 完整回放；公地只看 D+2，私人觀測所最多提早一個節點，且不見標的、方向、數量或價格。
3. **世界規模**：封測一座城市、50 位居民（10 高頻、40 低頻）與三種同權思考核心；契約從第一天支援更大公共世界。

## 最終架構 gate

外部嚴格審查的最終判決為 **「可進 pre-code P0」**，終局架構分數 **93／100**。不再需要重切定位、世界時間或 bounded context；但 feature code 前必須完成五項版本化交付：canonical event catalog、world-day saga、紙上執行規則、private／public disclosure matrix、simulation tier 與 50→50,000 scale proof。

這五項是世界可重播、可隔離與可擴張的地基，不是產品功能的縮減。完整可驗收條件見[技術架構 §7](./game-technical-architecture.md)與[Pre-code backlog](../BACKLOG.md)。
